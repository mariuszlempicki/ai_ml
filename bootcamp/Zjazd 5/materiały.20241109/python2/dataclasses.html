    Dataclasses

    Dataclasses to moduł wprowadzony w Pythonie 3.7 w celu ułatwienia tworzenia klas będących kontenerami danych. W tym celu dostarcza dekorator @dataclass.

    Kluczowe cechy:

    1. Automatyczne generowanie metod specjalnych: Automatyczne generowanie metod takich jak __init__(), __repr__(), i __eq__().

    2. Typowanie i domyślne wartości: Możliwość określania typów dla atrybutów i ustawiania domyślnych wartości.

    3. Porównywanie obiektów: Ułatwione porównywanie obiektów dzięki automatycznemu dodawaniu metod porównujących.

    4. Niemutowalność: Możliwość zdefiniowania dataclass jako obiektu niezmiennego (immutable) za pomocą parametru frozen.

    Proste przykłady

    1.

    from dataclasses import dataclass

@dataclass
class Point:
    x: int #  (1)!
    y: int

p = Point(1, 2)
    1. Na podstawie definicji pól i ich typów tworzone są metody specjalne

    2.

    from dataclasses import dataclass

@dataclass 
class POI:
    name: str
    lat: float
    lon: float

poi = POI('Rysy', 49.179562, 20.088062)

    Wartości domyślne

    from dataclasses import dataclass
from datetime import datetime
from typing import Optional

@dataclass
class Task:
    title: str
    description: str
    due_date: Optional[datetime] = None
    completed: bool = False

    Dodatkowe metody

    Wynikiem działania dekoratora @dataclass jest "zwykła" klasa w języku Python, można więc dodawać do niej dodatkowe metody.

    Przykład

    from dataclasses import dataclass
from datetime import datetime
from typing import Optional

@dataclass
class Task:
    title: str
    description: str = ''
    due_date: Optional[datetime] = None
    completed: bool = False

    def days_left(self):
        now = datetime.now()
        return (self.due_date - now).days if self.due_date is not None else None

task = Task('Buy milk', due_date=datetime(2023, 12, 23))
print(task)
print(f'days left: {task.days_left()}')

    Task(title='Buy milk', description='', due_date=datetime.datetime(2023, 12, 23, 0, 0), completed=False)
days left: 13

    Zaawansowana inicjalizacja pól klasy

    Specyfikator field() jest używany do indywidualnej konfiguracji każdego pola w klasie danych.

    Parametry, które przyjmuje field():

    • default: Domyślna wartość pola
    • default_factory: Bezargumentowa funkcja, zwracająca wartość początkową pola
    • init: Czy używać pola w metodzie .__init__()? (Domyślnie True.)
    • repr: Czy używać pola w reprezentacji obiektu przez repr? (Domyślnie True.)
    • compare: Czy używać wartości pola przy porównaniach? (Domyślnie True.)
    • hash: Czy używać pola przy obliczaniu wartości hash()? (Domyślnie używa tego samego ustawienia co compare.)
    • metadata: Mapa (słownik) zawierająca dodatkowe informacje o polu
    • kw_only: Czy pole może być inicalizowane tylko jako argument nazwany w __init__() (Domyślnie False)

    Przykłady

    from dataclasses import dataclass, field
from datetime import datetime
from typing import Optional

@dataclass
class Task:
    title: str
    description: str = field(default='', repr=False)
    due_date: Optional[datetime] = field(default_factory=datetime.today)
    completed: bool = False

    def days_left(self):
        now = datetime.now()
        return (self.due_date - now).days if self.due_date is not None else None

task1 = Task('Buy milk', due_date=datetime(2023, 12, 23))
task2 = Task('Pick up laundry')
print(task1)
print(task2)
print(f'days left: {task1.days_left()}')
print(f'days left: {task2.days_left()}')

    Task(title='Buy milk', due_date=datetime.datetime(2023, 12, 23, 0, 0), completed=False)
Task(title='Pick up laundry', due_date=datetime.datetime(2023, 12, 10, 4, 37, 37, 392913), completed=False)
days left: 8
days left: -1

    Parametry dekoratora @dataclass

    • init: Czy automatycznie generować metodę __init__() dla klasy. (Domyślnie True)

    • repr: Czy automatycznie generować metodę __repr__() dla klasy. (Domyślnie True)

    • eq: Czy automatycznie generować metodę __eq__() dla klasy. (Domyślnie True).

    • order: Gdy ustawione na True, automatycznie generuje metody porównujące (__lt__, __le__, __gt__, __ge__) dla klasy. (Domyślnie False).

    • unsafe_hash: Gdy ustawione na True, generuje metodę __hash__() nawet wtedy, gdy eq jest False lub gdy klasa jest mutowalna (frozen=False).

    • frozen: Gdy ustawione na True, tworzy niemutowalne (immutable) instancje dataclass. Próba zmiany atrybutów obiektu spowoduje wygenerowanie błędu. (Domyślnie False).

    • kw_only: Gdy ustawione na True, wszystkie pola klasy są wymagane jako argumenty nazwane (keyword-only arguments) w automatycznie generowanym konstruktorze __init__(). To znaczy, że muszą być przekazywane do konstruktora przy użyciu nazwy pola. (Domyślnie False).

    • match_args: Gdy ustawione na True, generuje specjalną krotkę __match_args__, która jest wykorzystywana w dopasowywaniu wzorców (pattern matching). (Domyślnie True).

    • slots: Jeśli ustawione na True, generuje automatycznie atrybut __slots__ dla klasy, co może poprawić wydajność poprzez oszczędność pamięci. (Domyślnie False).

    • weakref_slot: Jeśli ustawione na True, dodaje dodatkowy slot __weakref__ do klasy, umożliwiając obiektom klasy bycie celami słabych referencji. (Domyślnie False).

    Przykład

    from dataclasses import dataclass

@dataclass(eq=True, frozen=True, kw_only=True)
class Point:
    x: int
    y: int

    Metoda __post_init__

    Używanie Metody __post_init__ w Dataclassach Pythona

    Metoda __post_init__ w dataclassach Pythona jest specjalną metodą, która jest automatycznie wywoływana zaraz po zakończeniu inicjalizacji obiektu za pomocą metody __init__. Pozwala ona na wykonanie dodatkowej logiki lub inicjalizacji, które nie są bezpośrednio związane z prostym przypisywaniem wartości do pól klasy.

    Cechy metody __post_init__:

    1. Automatyczne Wywołanie: Metoda __post_init__ jest wywoływana automatycznie zaraz po __init__.

    2. Dostęp do Pól Klasy: Masz dostęp do wszystkich pól zdefiniowanych w dataclassie.

    Zastosowania:

    1. Dodatkowe obliczenia na podstawie wartości pól

    2. Weryfikacja, generowanie i przetwarzanie danych wejściowych

    Przykłady

    from dataclasses import dataclass

@dataclass
class Person:
    name: str
    age: int

    def __post_init__(self):
        if self.age < 0:
            raise ValueError("Age cannot be negative")
        self.name = self.name.title() 


person = Person("jan kowalski", 30)
print(person)

    Person(name='Jan Kowalski', age=30)

    from dataclasses import dataclass

@dataclass
class ServerInfo:
    ip: str
    hostname: str
    domain: str
    fqdn: str = field(init=False)

    def __post_init__(self):
        self.fqdn = f'{self.hostname}.{self.domain}'

server = ServerInfo('10.0.0.1', 'myhost', 'example.com')
print(server)

    ServerInfo(ip='10.0.0.1', hostname='myhost', domain='example.com', fqdn='myhost.example.com')

    Konwersja instancji dataclass

    Metody .asdict() i .astuple() w Dataclassach Pythona

    Dataclass w Pythonie oferuje dwie użyteczne metody: .asdict() i .astuple(), które pozwalają na łatwe konwertowanie instancji dataclass do słownika (dictionary) i krotki (tuple).

    Metoda .asdict()

    Metoda .asdict() konwertuje instancję dataclass na słownik. Klucze słownika odpowiadają nazwom pól w dataclass, a wartości są odpowiednimi wartościami tych pól.

    Przykład

    from dataclasses import dataclass, asdict

@dataclass
class Person:
    name: str
    age: int

person = Person("Jan Kowalski", 30)
print(asdict(person))

    {'name': 'Jan Kowalski', 'age': 30}

    Metoda .astuple()

    Metoda .astuple() konwertuje instancję dataclass na krotkę. Elementy krotki są wartościami pól w dataclass, ułożonymi w kolejności, w jakiej są zdefiniowane w klasie.

    from dataclasses import dataclass, astuple

@dataclass
class Person:
    name: str
    age: int

person = Person("Jan Kowalski", 30)
print(astuple(person))

    ('Jan Kowalski', 30)

    Metody .asdict() i .astuple() tanowią sposoby na konwersję instancji dataclass do standardowych typów Pythona, co jest szczególnie przydatne w kontekście serializacji danych i pracy z API webowym.
