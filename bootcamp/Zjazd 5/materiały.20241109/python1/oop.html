    Programowanie Obiektowe (podstawy)

    Obiektowość w języku Python

    • W Pythonie wszystko jest obiektem.
    • Obiektami są funkcje, klasy, bloki kodu itd.
    • Każdy obiekt w Pythonie ma tożsamość (ang. identity), typ (ang. type) i wartość (ang. value).

    Tożsamość

    • Tożsamość obiektu nigdy się nie zmienia.
    • Wbudowana funkcja id() zwraca liczbę całkowitą będącą tożsamością obiektu.
    • W interpreterze CPython id(x) zwraca adres zmiennej x.
    • Do porównania tożsamości dwóch obiektów służy operator is.

    Typ

    • Typ obiektu jest niezmienny.
    • Typ określa zachowanie obiektu.
    • Typ obiektu można sprawdzić przy pomocy funkcji type().
    • Typ jest także obiektem.

    Wartość

    • Wartość pewnych obiektów może ulegać zmianie:
      • Obiekty zmienne (ang. mutable) – ich wartość może być zmieniana
      • Obiekty niezmienne (ang. immutable) – ich wartość nie może być zmieniana
    • Niezmienne kontenery mogą zawierać referencje do zmiennych obiektów.

    Obiekty

    • Obiekty są zbiorem zmiennych i funkcji opakowanych w jedną encję.
    • Zmienne w obiektach nazywamy atrybutami obiektu.
    • Funkcje w obiektach nazywamy metodami obiektu.
    • Dostęp do metod i atrybutów obiektu odbywa się poprzez operator . (kropka).

    Klasy w języku Python

    • Klasy są definicjami obiektów.
    • Zawierają metodę tworzącą i inicjalizującą nowe obiekty.
    • Obiekty często nazywamy instancjami klasy.

    Definicja i tworzenie klas

    class MyClass:
    pass

obj = MyClass()

    Stworzenie instancji klasy odbywa się przez wywołanie jej nazwy

    Metoda __init__()

    Po stworzeniu instancji klasy automatycznie wywoływana jest metoda __init__(). Służy ona do inicjalizacji nowo utworzonego obiektu np. poprzez ustawienie wartości początkowych jego atrybutów. Wywoływana jest automatycznie, a argumenty, które przyjmuje powinny być przekazane do konstruktora w momencie tworzenia klasy.

    class Car:
    def __init__(self, make, model, prod_year):
        self.make = make
        self.model = model
        self.prod_year = prod_year


mycar = Car('Toyota', 'Yaris', 2000)

    Atrybuty

    Atrybuty używane są do przechowywania danych o cechach obiektu. Atrybutami dynamicznymi nazywamy te atrybuty, które przynależą do obiektu, a atrybutami statycznymi te które tworzone są na poziomie klas.

    class Car:
    wheels = 4  #  (1)!
    classic_age = 25

    def __init__(self, make, model, prod_year):
        self.make = make  #  (2)!
        self.model = model
        self.prod_year = prod_year


mycar = Car('Toyota', 'Yaris', 2000)

print(f'{mycar.make=}')
print(f'{mycar.model=}')
print(f'{mycar.prod_year=}')
print(f'{mycar.classic_age=}')
print(f'{Car.classic_age=}')
    1. Atrybuty statyczne
    2. Atrybuty dynamiczne

    Metody

    Metody operują na danych klasy. Zawsze pierwszym argumentem każdej metody jest instancja obiektu danej klasy. Według konwencji argument ten nazywany jest self. Podawany jest jawnie przy definicji metody natomiast podczas wywołania uzupełniany jest automatycznie. 

    class Car:
    wheels = 4  #  (1)!
    classic_age = 25

    def __init__(self, make, model, prod_year, color):
        self.make = make  #  (2)!
        self.model = model
        self.prod_year = prod_year
        self.color = color

    def repaint(self, new_color):
        self.color = new_color


mycar = Car('Toyota', 'Yaris', 2000, 'blue')

print(f'{mycar.make=}')
print(f'{mycar.model=}')
print(f'{mycar.color=}')
mycar.repaint('red')
print(f'{mycar.color=}')

    Metody i atrybuty prywatne

    W Pythonie metody i atrybuty do użytku wewnętrznego rozpoczynają się od pojedynczego znaku podkreślenia. Jest to jedynie konwencja służąca do oznaczania elementów stanowiących niepubliczny interfejs obiektu.

    class Car:
    wheels = 4  #  (1)!
    classic_age = 25

    def __init__(self, make, model, prod_year, color, mileage):
        self.make = make  #  (2)!
        self.model = model
        self.prod_year = prod_year
        self.color = color
        self._mileage = mileage

    def repaint(self, new_color):
        self.color = new_color

    def get_mileage(self):
        return self._mileage

    def set_mileage(self, mileage):
        if mileage >= self._mileage:
            self._mileage = mileage
        else:
            print("Error: Mileage cannot be decreased")

    def _format_car_details(self):
        return f'{self.make} {self.model} {self.prod_year}'

    def get_car_info(self):
        return self._format_car_details()


mycar = Car('Toyota', 'Yaris', 2000, 'blue', '120000')

print(f'{mycar.make=}')
print(f'{mycar.model=}')
print(f'{mycar.color=}')
mycar.repaint('red')
print(f'{mycar.color=}')

    Metody specjalne

    • Python zawiera specjalne zmienne, funkcje oraz metody, służące różnym celom.
    • Ich nazwy tworzone są według wzoru __nazwa__

    • Klasy w języku Python implementują pewne operacje przy pomocy metod specjalnych:

      • Operacje arytmetyczne
      • Dostęp do atrybutów
      • Obsługa indeksów i slice

    • object.__repr__(self)

      • Wywoływana przez wbudowaną funkcję repr().
      • Zwraca reprezentację obiektu.
      • Jeśli to możliwe powinna zwracać wyrażenie w języku Python pozwalające na stworzenie danego obiektu.
      • Jeśli nie jest to możliwe to powinna zwrócić informację w formacie <opis obiektu>

    • object.__str__(self)

      • Zwraca “nieoficjalną” reprezentację obiektu jako łańcuch znaków.
      • Zwracana wartość nie musi być poprawnym wyrażeniem w języku Python.
      • Zwracana wartość musi być łańcuchem znaków.
    class Car:
    wheels = 4  #  (1)!
    classic_age = 25

    def __init__(self, make, model, prod_year, color, mileage):
        self.make = make  #  (2)!
        self.model = model
        self.prod_year = prod_year
        self.color = color
        self._mileage = mileage

    def repaint(self, new_color):
        self.color = new_color

    def get_mileage(self):
        return self._mileage

    def set_mileage(self, mileage):
        if mileage >= self._mileage:
            self._mileage = mileage
        else:
            print("Error: Mileage cannot be decreased")

    def _format_car_details(self):
        return f'{self.make} {self.model} {self.prod_year}'

    def get_car_info(self):
        return self._format_car_details()

    def __str__(self):
        return f"<{self._format_car_details()}>"


mycar = Car('Toyota', 'Yaris', 2000, 'blue', '120000')

print(mycar)

    @classmethod i @staticmethod

    • @classmethod
      • Reprezentuje metodę klasową
      • Pierwszym argumentem jest klasa, a nie instancja jak w zwykłej klasie
      • Zastosowania:
        • Tworzenie fabryk instancji
        • Dostęp do atrybutów klasowych
    • @staticmethod
      • Reprezentuje metodę statyczną, zasadniczo jest to zwykła funkcja umieszczona w ciele klasy
      • Nie przyjmuje argumentu będącego ani instancją, ani klasą
      • Zastosowania
        • Grupowanie funkcji związanych logicznie z klasą, ale nie korzystających z jej atrybutów i metod

    Zadanie

    Zaprojektuj klasy mające przechowywać dane o rachunkach. Rozwiązanie powinno zawierać:

    Klasę reprezentującą rachunek

    Rachunek powinien zawierać:

    • numer
    • datę wystawienia
    • listę pozycji
    • rabat

    Klasa przechowująca dane rachunku powinna posiadać następujące metody:

    • metoda pozwalająca kolejne pozycje z rachunku wraz z ich numerami kolejnymi (np. w formie krotek (1, ))
    • metoda obliczająca sumę z wszystkich pozycji i uwzględniająca rabat jeśli jest zdefiniowany
    • metoda zwracająca liczbę pozycji na rachunku

    Klasę reprezentującą pozycję na rachunku

    Atrybuty:

    • nazwa artykułu
    • ilość
    • cena jednostkowa

    Metody:

    • metoda zwracająca wartość pozycji (na podstawie ceny i ilości)
